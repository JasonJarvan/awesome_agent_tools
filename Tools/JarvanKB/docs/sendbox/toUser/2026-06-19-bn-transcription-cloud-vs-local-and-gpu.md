# BN 转录(云/本地)+ 本机 GPU 能否启用 — 结论（2026-06-19）

> 给你的说明信。回应:本机 BN 转录怎么实现(云/本地、是否用显卡)+【让 GPU 可用】的步骤/成本/能否全自动不重启。

## 一、BN 转录现状
- **= 云端 bcut(B站必剪免费 ASR)**,不吃本机算力、不用显卡(`TRANSCRIBER_TYPE=bcut`,容器内权威 `transcriber.json` 已确认)。
- **字幕优先**:视频有字幕 → 直接用,**零转录算力**(我们引擎还会主动预取 B站字幕,需 SESSDATA);**无字幕才回落 bcut 云 ASR**。
- ⚠️ 走 ASR 那条时,**视频音频会上传到 B站 bcut 服务器**(对 B站来源内容算低敏感,但确实离开本机;免 key、不传 cookie)。

## 二、能不能让本机 GPU 可用?——结论:**不值得,保持云 bcut**
你的判据是「**可全自动(无人)+ 不重启** → 直接做;否则只出报告」。实测 → **不满足全自动**,故按约定**不派 impler**,只出本报告。

**三点关键(含对我先前预判的纠正)**:

1. **做不到全自动**(硬拦路):修驱动 / 装 nvidia-container-toolkit / 重启 dockerd 每步都要 root,而本机**没有免密 sudo**(`sudo -n true` 要密码)→ **必须你手动输密码**,无人无法完成。

2. **不重启机器 OK**(纠正我之前"多半要重启"的预判):
   - **580 驱动其实支持 Maxwell 860M**(580 正是保留 Maxwell 的最后 legacy 分支,不是被砍)——我先前担心的"需降级驱动=重启级"是错的;
   - 真因只是**内核模块版本错位**:加载的是 `580.126.09`(对应当前内核 `6.17.0-20`),用户态库已升到 `580.159`(其模块只装在另一个没启动的内核 `6.17.0-35` 上);
   - 修复**不需重启机器**:显示器走 Intel 核显、独显没被任何进程占用(Optimus on-demand)→ 可在线重载/重装匹配模块,或改启另一个内核。

3. **但即使修好也不值得**(更深的拦路虎):
   - 现成的 `ctranslate2`(faster-whisper 的推理引擎)CUDA 包**在 CC 5.0 的 Maxwell 上直接报错** `cudaErrorNoKernelImageForDevice` → 必须**从源码重编 CT2**(开 CC 5.0)才能跑;
   - 2GB 显存只够 tiny/base/small,且 < CC 7.0 **不支持 FP16** → 这块 2014 老卡相比 CPU 提升有限;
   - 本机 `/` 只剩 **12GB**(已用 69%),CUDA 镜像好几个 GB,紧张;
   - **对比免费、零维护、够快、且只在无字幕时才触发的云 bcut,性价比为负。**

## 三、建议
**保持现状(云 bcut)。不修 GPU、不派 impler。**
重启这个评估的两个触发点:① 你**换显卡**;② 你确实需要**完全本地/离线转录**(为避免音频上云)。届时主要工作 = 你授权 sudo + 修内核模块错位(免重启)+ 装 toolkit + **从源码编 CT2** + 换 CUDA 镜像。
